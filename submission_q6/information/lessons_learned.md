# 经验教训 - Q6: 硬币的另一面

## 关键决策点

### 1. 数据预处理
- **决策**: 去掉高度相关的MPI和La特征
- **原因**: 相关性>0.90会导致多重共线性
- **结果**: 模型稳定性提升

### 2. 特征选择
- **决策**: 使用Random Forest的特征重要性选择Top 4
- **Top 4特征**: num_O, TPSA, HBA, HOMO
- **物理解释**: 
  - num_O: 氧原子数 → Pb²⁺配位结合点
  - TPSA: 拓扑极性表面积 → 氢键能力
  - HBA: 氢键受体数 → 表面相互作用
  - HOMO: 最高占据分子轨道 → 电负性/静电吸引

### 3. 模型选择
- **决策**: 使用5种分类器对比
- **发现**: Random Forest和Logistic Regression表现最佳
- **洞察**: 简单模型在此任务上足够有效

## 遇到的问题与解决

### 问题1: XGBoost安装
- **现象**: xgboost包安装失败
- **解决**: 使用`pip install xgboost --break-system-packages`

### 问题2: 特征缩放
- **现象**: SVC对特征尺度敏感
- **解决**: 使用StandardScaler标准化

## 最佳实践

1. **数据探索优先**: 先理解数据分布和特征相关性
2. **简单模型优先**: 从LR和RF开始，再尝试复杂模型
3. **交叉验证**: 使用StratifiedShuffleSplit保持类别平衡
4. **可视化**: 绘制ROC曲线和特征重要性帮助理解
